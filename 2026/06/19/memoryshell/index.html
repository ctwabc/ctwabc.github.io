tomcat内存马

因为在tomcat中资源的分配加载都跟standardcontext有关，所以说内存马就是找到standardcontext，通过standardcontext将恶意的listener，filter，servlet，valve放进去。

1.listener型：

有各种不同的listener，其中ServletRequestListener无疑是最好用的。因为它监听的是ServletRequest，访问web网站的任意位置都会触发这个监听器。

listener存储在applicationEventListenersList，调用standardcontext.addApplicationEventListenersList就好了。因为是tomcat+jsp所以说这里有一个通用的方式去获取standardcontext

ServletContext servletContext = request.getServletContext();

Field applicationContextField = servletContext.getClass().getDeclaredField(“context”);

applicationContextField.setAccessible(true);

ApplicationContext applicationContext = (ApplicationContext) applicationContextField.get(servletContext);

Field stantardContextField = applicationContext.getClass().getDeclaredField(“context”);

stantardContextField.setAccessible(true);

StandardContext context = (StandardContext) stantardContextField.get(applicationContext);

2.filter型:

首先需要对filter的调用流程需要一些了解。

filter.doFilter->ApplicationFilterChain.internalDoFilter->filter.dofilter

tomcat的filterchain是在每一次的请求中重新组装的

在ApplicationFilterChain中

从ApplicationFilterConfig获取filterconfig从filterconfig获取filter

这是filter的调用流程

现在需要关心的是filter是怎么放进去的

关键就是把filterconfig放到ApplicationFilterChain里面

ApplicationFilterFactory.createFilterChain在这个方法里面有filterChain.addFilter(filterConfig)

现在就是需要走到这里了。

这里涉及到几个概念

filtermaps:我们都知道filter是过滤器，既然是过滤器，那就需要知道在访问哪些地方的时候调用哪个过滤器，他就是干这个工作的。

filterconfig:对filter的封装，除了filter还有别的一些数据，他是使用filterdef创建的

filterdef:对于filter的静态定义，用于创建filter。里面记录了 Filter 的名字（filterName）、Filter 对应的实现类（filterClass），以及初始化参数（initParams）

dispatcher:决定了请求以什么方式到达的时候会调用。

REQUEST（最常用）： 只有当客户端直接发送 HTTP 请求访问这个 URL 时，才触发 Filter。

FORWARD： 当后端代码通过 request.getRequestDispatcher(“/target”).forward(request, response) 进行内部转发时，才触发 Filter。

INCLUDE： 当页面通过 include 包含其他页面时触发。

ERROR： 当系统发生错误、跳转到错误处理页面（404/500）时触发。

ASYNC： 异步请求时触发。

1.遍历filtermaps获得filtermap，将与这次请求的dispatcher 类型以及url路径相匹配的filterconfig从filterconfigs中拿出来，放入filterchain。

2.遍历filtermaps获得filtermap，将与这次请求的dispatcher 类型以及servlet名字相匹配的filterconfig从filterconfigs中拿出来，放入filterchain。

所以说我们得先把filtermap放入filtermaps中再把filterconfig放入filterconfigs中，而filterconfig需要filterdef。这个filterdef也需要放在context中，主要是为了对账，防止检测出没有filterdef但是有filterconfig这种情况。
