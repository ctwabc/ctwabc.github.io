log4j2

发表于 2026-06-17 更新于 2026-06-19 分类于 Java安全
本文字数： 1.1k

Log4j2 是 Java 里最常用的日志框架。为了让程序员在打印日志时更方便，Java 官方在 Log4j2 中加入了一个叫 Lookups（查找替换） 的高级功能。

它允许你在日志里写一个特殊占位符 ${...}，Log4j2 打印时会自动把它解析并替换掉。例如：

  • 打印当前系统版本：logger.info("OS: ${java:os}"); $\rightarrow$ 输出 OS: Windows 11
  • 打印某个环境变量：logger.info("Path: ${env:USER}"); $\rightarrow$ 输出 Path: root

到这里为止都很正常。但致命的是，Log4j2 开发者为了追求极致的灵活性，竟然把 JNDI 协议也接了进去！

也就是说，如果在日志里写 ${jndi:rmi://...}，Log4j2 底层就会无条件地去调用 JNDI 接口。
